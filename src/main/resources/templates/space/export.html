导出到个人博客

下载后会得到一个 ZIP，里面包含可直接部署到 GitHub Pages 的静态博客页面、原始 Markdown 和元数据。成员毕业后可以直接把自己的文章带走，不依赖实验室内网。

下载导出包 完善个人资料 查看项目源码

导出包包含什么

  • 静态首页 `index.html`，上传到 GitHub 仓库即可作为个人博客首页。
  • 每篇文章对应的 `posts/*.html` 页面，可直接访问。
  • `markdown/*.md` 原始 Markdown 文件，方便后续迁移到 Hexo、Hugo、VuePress 等系统。
  • `metadata/profile.json` 和 `metadata/posts.json`，方便做二次导入。
  • 导出包不包含密码、邮箱、评论和其他成员的数据。

推荐迁移步骤

  • 先在 GitHub 创建个人博客仓库，例如 `username.github.io`。
  • 下载并解压当前导出包，把全部文件放到仓库根目录。
  • 在 GitHub Pages 中启用 `main` 分支根目录发布。
  • 如果你未来换博客框架，直接继续使用导出的 Markdown 和元数据即可。

当前导出配置
